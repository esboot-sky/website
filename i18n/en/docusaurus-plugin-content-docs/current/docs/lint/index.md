# Rules and Lint

To keep coding conventions consistent, ESBoot ships with built-in script and style rules together with lint checks.

- script linting is based on [ESLint](https://eslint.org/)
- style linting is based on [Stylelint](https://stylelint.io/)
- commit message checks are based on [Commitlint](https://commitlint.js.org/)
- pre-commit checks are enforced through [Husky](https://github.com/typicode/husky)

All related config files are normally generated during installation. You can also run `esboot prepare` manually.
