`ESBoot` implements internationalization on top of [React Intl](https://github.com/formatjs/react-intl).

In most real projects, you will at least support Simplified Chinese and Traditional Chinese, and often English as well.

## What Not To Do: TypeScript-Based Language Files

Historically, teams often created nearby `lang` folders containing files such as:

- `zh-CN.ts`
- `zh-TW.ts`
- `en-US.ts`

This becomes hard to maintain because TypeScript is too flexible: files can import each other, include logic, and gradually scatter translations across the codebase.

## Recommended: JSON-based Language Files

JSON keeps translations simpler and easier to audit.

Benefits:

- less flexibility, so fewer accidental patterns
- fewer files and less duplication
- easier linting and key consistency checks

## Directory Layout

In ESBoot, i18n files are expected under `src/lang`:

```plaintext
src/
  ├── lang/
  │   ├── zh-CN.json
  │   ├── zh-TW.json
  │   ├── en-US.json
```

This makes linting and key validation much more predictable.

## Translation Keys

### Usage

```tsx
import { FormattedMessage } from 'react-intl';

<FormattedMessage id="global.project" />
```

### Naming Convention

Use snake_case or hyphen-separated naming, not camelCase, not uppercase, and not Chinese.

Best practice:

- top-level keys represent pages or domains, so kebab-case is often a good fit
- deeper keys should use words or snake_case names

## langJsonPicker and On-demand Loading

When [useLangJsonPicker](../config#uselangjsonpicker) is enabled, entry files can specify which key groups to load.

```ts
export default {
  langJsonPicker: ['global', 'a'],
};
```

This keeps language payloads smaller, especially in large MP projects with many pages.

## VS Code Plugin

The [i18n Ally](https://marketplace.visualstudio.com/items?itemName=Lokalise.i18n-ally) extension makes translation keys much easier to inspect and edit.

## Summary

- keep all language files in `src/lang`
- use layered keys like `xx.xx.xx`
- enable `useLangJsonPicker` in MP projects
- keep naming consistent and machine-checkable
